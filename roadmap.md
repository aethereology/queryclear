# Roadmap — queryclear

Smallest sellable thing first. Each phase ends with something we can sell or use.

## Phase 0 — Foundation ✅ (done 2026-05-29)

- Fixed dangerous home-dir git root; Aethelo work backed up safely
- Clean queryclear repo tracking GitHub
- Strategy, structure, and project docs in place

## Phase 1 — Sellable MVP (current)

**Goal: be able to close a client.**
1. Landing page (hero → offer → How It Works → deliverables → CTA), per `UI_direction.md`
2. Accessible lead-capture form (fields in `product_spec.md`)
3. **One real sample GEO audit** (built against the `seed_data.md` demo business) — the sales asset
4. Our own site is itself GEO-perfect: schema, `llms.txt`, sitemap, robots, OG, fast, a11y
5. Deploy (Vercel) at queryclear.com

**Exit:** a stranger can land, understand the offer, and book an audit; we have a
sample audit to show on the call.

## Phase 2 — Repeatable fulfillment

- Reusable audit template + report generator
- AI-visibility prompt runner (from `prompts.md`)
- Client intake → audit → proposal flow
- Page/schema/`llms.txt` templates for client builds

**Exit:** delivering a client audit/build is mostly repeatable, not bespoke.

## Phase 3 — Aethelo automation expansion

- Upsell path from queryclear clients into Aethelo automation systems
- Productize the parts of fulfillment that are still manual

## Explicitly NOT now

No SaaS product, no dashboard app, no client login, no billing platform until a
client is paying and asking for it. Service business first.
